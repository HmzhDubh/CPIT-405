CSS: Cascading Styling Sheet

Introduction Fundamentals

Introduction

CSS is a language to specify the presentation and the styling of the HTML document

And it is dedicated to define visual styling like font, color, layout, and animation, etc

Cascade: is the core of an algorithm to solve conflicts as multiple rules applied to a HTML element

CSS supports is inconsistent between browsers

Fundamentals

Basic Structure:

referencing CSS to be applied can be done in Three locations:

  1. Internal: In the head section using the style tag
  2. Inline: Within the html Element that is meant to be styled passed as an attribute to it ie, style="..."
  3. External: A CSS extension file includes all styling and referred to it in the head in a link element, link uses rel, href, and media ie, "print" attributes to refer to the styling sheet.
  4. Browser-Default stylesheet:
    • web browser may apply default styling when rendering the page
    • each browser has set of default styles and vary from one to another
    • Dev override these default styles by specifying their own styles
  5. User Stylesheet:
    • some browsers provide the ability for users to customize their own styles
    • It can affect browser stability and security

Cascading order algorithm :

Algorithm Steps:
  1. Relevance: All rules filtered from different sources and keep rules apply to a given element amd appropriate media
  2. Origin And Importance: ie, important flag
  3. Specificity: specificity of a rule when chossing one value or another
  4. Scooping Proximity: styles defied closer to an element have more influence that the further away ie, inline then internal then external and finally the browser default
  5. Order OF Appearance: if multiple rules with same power and location the last one will be applied

CSS Selectors:

  1. Universal Selector: " * " for all elements in an HTML documnet
  2. Element Selector: select an HTML element like h1 or p etc
  3. ID Selector: specify the element with the unique id match it starting with a " # " ig. #nav-bar
  4. Class Selector: specify elements with the class starting with a " . " ig. .header
  5. Attribute Selector: specify an element with a specific attribute ie, img[alt]
    6. Selector Description Example
    [attr] Match the element that have attr attribute declared input[required]
    [attr=value] Match the element that have attr attribute declared whose value is exactly = value input[type="password"]
    [attr~=value] Match the element that have attr attribute declared whose value is exactly = value OR the value is in the list of values that are space separated p[class~=special]
    [attr^=value] Match the element that have attr attribute declared whose values list Begins with exactly = "value" p[class^=box]
    [attr$=value] Match the element that have attr attribute declared whose values list Ends with exactly = "value" p[class^=box]
    [attr$=value] Match the element that have attr attribute declared whose values list Contains anywhere "value" within the value list p[class^=box]
  6. Combinator Selector: combine multiple elements in a single selector
    • 1- Descendant Selector: selector1 selector2: all elements inside the first selectors that matches the second are selected
    • 2- Child selector: element1 > element2: all direct children of the first selector that matches the second one are selected
    • 3- General Sibling: selector2 ~ selector2: all siblings of the first elements that matches the second one are selected
    • 4- Adjacent Sibling: selector1 + selector2: the immediate sibling of the first element is selected
  7. Pseudo-Classes
    • Syntax: selector:pseudo-class {style properties}
    • It indicates a special state of that selected element ie, hover, clicked on
      • Pseudo-class Description
      :visited Visited link
      :hover mouse only hovered on a link
      :required form element is required
      :optional form element is optional
      :first/last/nth-child first or last or the nth-child(ie, 5) child of the selector element
  8. Pseudo-Elements
    • ::first-line : style the first line of the block line element

      The length of the first line depends on the width of the element, or the document and the font size

    • ::after : add last child or new content to the selected element
    • ::before : add first child or new content at the beginning to the selected element
    • ::selection : style element that has been highlighted by the user

CSS Box Model

  • Fundamental concept in css to define how elements rendered on the screen
  • Each element is represented as a rectangle box, box contain Content, Padding, Border and, Margin
  • The content: The actual content of the element
  • The padding: The space between the content and the border
  • The border: The line around the element (padding and content)
  • The Margin: The space around the border
