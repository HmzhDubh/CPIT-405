Protocols of the web

protocols manage how browsers(clients) and servers communicate by enabling data transfer and render websites

It is important to understand protocols to develop secure and efficient and usable web applications

HTTP

  • HTTP: is a foundation of communication between browsers and servers
  • Request: is what browser use to ask for a resource or information necessary to render the web page
  • - which means the server doesn't store information about clients between different requests
  • HTTP is "stateless" protocol which means requests runs independently of other requests
  • HTTP request contain:
    • HTTP version type eg, HTTP/1, HTTP/2, HTTP/3
      • HTTP/1: uses single TCP connection per request aka sequential communication, which leads to Limited multiplexing: connetction can handle multiple requests but not efficiently
      • HTTP/2: Extends use of connection by multiplexing concurrent req/responds via single connection, It also uses more efficient dota transfer bin format and compress the headers to reduce data transfer
      • HTTP3: uses QUIC and UDP protocols QUIC: transfer protocol designed for mobil-heavy internet usage. UDP:provide faster connection and performance. They both address netwrok switching problrms and security by built-in encryption methods
    • URL
      • Uniform Resource Locator: reference for and address of a resource on the internet
      • URI: uniform Resource Identifier: a broader concept used to name the resources, It doesn't necessarily locate the resource
      • URL Components:
        • Protocol: ie, HTTP, HTTPS, FTP
        • Domain Name: human-readable name
        • Port: of the server the client should contact ie, 80, 443
        • Path: specific resource path on the server
        • Query String: to include additional data to be sent to the server Starts with ? symbol
        • Fragment: spicify an element of html starts by a hash #
    • HTTP Method like GET, POST,PUT, etc used to indicate an action to be performed by the server
    • Request headers And optional body
      • Headers: Are additional data for context and instructions to enable a better understanding between the client and the server
      • Common headers:
        • Authorization: contain credentials to auth user agent with server
        • Content-type: media type of the body of the request
        • User-agent: app type, os of client
        • Cookies: stored files sent by the server previously
      • Body
        • Actual data to be sent to the server ie. Login Data
        • Format can be JSON, XML or binary any format client and server agree to use
        • it is optional not all request contain body
    • HTTP Status Codes
      • 100s : Informational As the request is received and continue processing it
      • 200s : Successfully received and understood, and accepted ig. 200 OK
      • 300s : Redirection, further action will be performed to complete request
      • 400s : Client Error, client sent bad syntax or request cannot be fulfilled ig. 404 NOT FOUND
      • 500s : Server Error, server failed to complete a valid request ig. 500 SERVER INTERNAL ERROR

HTML

HyperText Markup Language: basic building block of web pages, uses <> tags to display content accordingly.

The purpose is to describe the structure of the web page by defining headers, paragraphs, lists, links, imgs, tables and many more.

Standardized by W3C and maintained by WHATWG at https://html.spec.whatwg.org

  • HTML Elements

Inline Elements:

  1. Don't start a new line
  2. Cannot contain Block elements
  3. Avoid using block elements within inline
  4. it may be used to format text within a block element
  5. EXAMPLES: a, span, input, button, label, and code

Block Elements:

  1. Start a new line
  2. Can contain inline elements
  3. it can use inline element
  4. EXAMPLES: a, span, input, button, label, and code

Void Elements:

  • Elements with no closing tags, it's not required but allowed to have it
  • they should be self-contained and not containing any child elements
  • They typically represent content inserted into doc inline such as img, meta, hr, input, link, area and embed, etc
